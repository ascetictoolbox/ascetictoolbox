COMPSs framework installation guide



1. Debian-based distributions

* Note:
- The following commands require root privileges and Internet connection.
- Once the packages are installed, please log out and back in again to complete the installation.

1.1 Add COMPSs package repository

1.1.1 Edit repository list

      • x86-64: echo "deb [arch=amd64] http://compss.bsc.es/rc/repo/debs stable main" >> /etc/apt/sources.list

      • No arch: echo "deb [arch=all] http://compss.bsc.es/rc/repo/debs stable main" >> /etc/apt/sources.list

1.1.2 Refresh package list

      • apt-get update

1.2 Install COMPSs packages

1.2.1 Java applications: COMPSs framework

      • apt-get install compss-framework

1.2.2 Python applications: COMPSs Python binding

      • apt-get install compss-python-binding

1.2.3 C/C++ applications: COMPSs C/C++ binding

      • apt-get install compss-c-binding

1.2.4 Tracing system (optional): COMPSs extrae

      • apt-get install compss-extrae



2. RedHat-based distributions

* Note:
- The following commands require root privileges and Internet connection.
- Once the packages are installed, please log out and back in again to complete the installation.

2.1 Add COMPSs package repository

2.1.1 Edit repository list

      • x86-64

        (Suse) zypper addrepo -f http://compss.bsc.es/rc/repo/rpms/stable/x86_64 compss

        (RHL, Centos) wget http://compss.bsc.es/rc/repo/rpms/stable/compss-x86_64.repo -O /etc/yum.repos.d/compss-x86_64.repo

      • No arch

        (Suse) zypper addrepo -f http://compss.bsc.es/rc/repo/rpms/stable/noarch compss

        (RHL, Centos) wget http://compss.bsc.es/rc/repo/rpms/stable/compss-noarch.repo -O /etc/yum.repos.d/compss-noarch.repo

2.1.2 Refresh package list

      • (Suse) zypper refresh

        (RHL, Centos) Nothing to do

2.2 Install COMPSs packages

2.2.1 Java applications: COMPSs framework

      • (Suse) zypper install compss-framework

        (RHL, Centos) yum install compss-framework

2.2.2 Python applications: COMPSs Python binding

      • (Suse) zypper install compss-python-binding

        (RHL, Centos) yum install compss-python-binding

2.2.3 C/C++ applications: COMPSs C/C++ binding

      • (Suse) zypper install compss-c-binding

        (RHL, Centos) yum install compss-c-binding

2.2.4 Tracing system (optional): COMPSs extrae

      • (Suse) zypper install compss-extrae

        (RHL, Centos) yum install compss-extrae



3. Additional setup

3.1 Configure SSH keys

By default, COMPSs uses SSH libraries for communication between nodes. Consequently, after COMPSs is installed on a set of machines, the SSH keys must be configured on those machines so that COMPSs can establish passwordless connections between them. This requires to install the OpenSSH package (if not present already) and follow these steps:

3.1.1 Generate an SSH key pair

      • ssh-keygen -t dsa

3.1.2 Distribute the public key (~/.ssh/id_dsa.pub) to all the machines and configure it as authorized

      • (On every machine) cat id_dsa.pub >> ~/.ssh/authorized_keys

3.1.3 Check that passwordless SSH connections are working

      • (Between any pair of machines) ssh machineX

3.2 Deploy the COMPSs task monitor

3.2.1 Deploy the WAR package to applications container:

    • Copy /opt/COMPSs/Monitor/compss-monitor.war to your web applications container (webapps/):

    • Example (case of Debian Tomcat 6 installation):

      • Copy /opt/COMPSs/Monitor/compss-monitor.war to /var/lib/tomcat6/webapps/

      • Add export IT_MONITOR=$HOME line to /usr/share/tomcat6/bin/catalina.sh file.

    • Example (case of CentOS Tomcat 6 installation):

      • Copy /opt/COMPSs/Monitor/compss-monitor.war to /var/lib/tomcat6/webapps/

      • Add export IT_MONITOR=$HOME line to beginning of /usr/sbin/tomcat6 file.

3.2.2 Export IT_MONITOR environment variable:

    • Export the environment variable for the applications container:

      • export IT_MONITOR=$HOME (this path should point to the $HOME of the user which calls the application).
